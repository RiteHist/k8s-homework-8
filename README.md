## Расчёт рабочих нод и кластера

- Control Plane ноды (3 шт для HA): резервируем 2–4 ядра и 8–16 ГБ ОЗУ на каждую (стандартные рекомендации Kubernetes).  
- Рабочие ноды: на каждой резервируем ~0.5–1 ядро и 1–2 ГБ под kubelet, system pods и eviction threshold.

На основе проведённых расчётов, в качестве ответа предлагается использовать кластер с 3 control plane нодами (по 4 ядра и 16 ГБ ОЗУ каждая) и 4 рабочими нодами (по 8 ядер и 32 ГБ ОЗУ каждая). Это обеспечивает полный запас мощности даже при падении одной ноды, учитывает служебные ресурсы и оставляет место для роста
